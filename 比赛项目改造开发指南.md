# 基于低空无人机视觉的桥梁表观病害精细化智能检测平台 - 改造开发指南

## 一、 项目背景与改造目标

本项目原为一个“无人驾驶路况识别巡检小车”系统，现需针对**“基于低空无人机视觉的桥梁表观病害精细化智能检测算法开发”**比赛进行整体前端改造。
核心任务是**将前端界面的业务场景从“道路巡检”切换为“无人机桥梁巡检”**，并且要在不接入真实 AI 后端的前提下，**通过 Mock 数据高保真地模拟出比赛要求的复杂 AI 视觉能力（尤其是像素级分割与量化能力）**。

最终产出的 Web 界面必须能作为一个专业、极具科技感的政府/工程级 GIS 系统来进行路演展示。

---

## 二、 核心业务术语与模型设定（必须严格遵守）

为了体现比赛要求的专业性，UI 和模拟数据中必须使用以下设定：

1. **病害类型（至少涵盖以下 3 种以上）**：
   - 混凝土裂缝 (Concrete Crack)
   - 剥落/掉块 (Spalling)
   - 钢筋裸露 (Exposed Rebar)
   - 钢结构锈蚀 (Steel Corrosion)
   - 泛碱/渗水 (Efflorescence/Seepage)
2. **量化指标（比赛核心得分点）**：
   - AI 识别结果不仅要有**类别**和**置信度 (Confidence)**，更要具备**物理量化属性**（如：裂缝最大宽度 2mm、长度 1.5m，剥落面积 0.5m² 等）。
3. **视觉展示方式**：
   - 从原有的单纯矩形框 (Bounding Box) 升级为 **像素级多边形分割掩膜 (Polygon Mask)**，以体现比赛要求的“实例分割快、准、细”特点。

---

## 三、 页面架构与视觉重构指引

采用 React + TailwindCSS + Lucide Icons 构建，保持深色科技感（Dark GIS 风格）的主色调。

### 1. 全局导航 (Navigation)
- **系统名称**：更改为“低空无人机桥梁病害智能检测平台”。
- **侧边栏结构精简与重命名**：
  - **驾驶舱 (Dashboard)**: 整体无人机巡检网概览（起飞架次、桥梁健康度总览）。
  - **无人机巡检 (UAV Inspection)**: 包含“巡检地图”与“实时图传/作业视频” (核心功能页)。
  - **病害管理 (Disease Mgmt)**: 结构化病害台账、量化分析报表。

### 2. 核心演示页：桥梁 AI 智能巡检工作台 (The Inspection Workspace)
此页面是比赛演示的“杀手锏”，布局建议采用**左右分栏或经典的大屏监控布局**。

#### A. 左/中主舞台区：无人机视觉图传与 AI 叠加 (AI Overlay)
- **视觉源**：播放一段无人机环绕桥梁拍摄的视频或高清图像组。
- **AI 动态开关 (AI ON/OFF)**：
  - 必须提供显眼的“AI 识别”热切换开关或“原图/AI分屏”对比效果。这是路演时给评委展现“AI 价值”的最好方式。
- **Overlay 渲染技术**：
  - 在视频/图片上方使用 Canvas 或 SVG 渲染多边形 Masks（半透明填充色 + 高亮发光边缘）。
  - **标签**：Mask 旁边悬浮跟随的 Tag，显示：`钢筋裸露 · 96%`。
  - **动态平滑**：框线和掩膜的出现过程要有淡入效果。

#### B. 右侧面板：实时智能分析与量化列表
- 随视频时间戳或图片切换，实时滚动命中当前画面的病害列表。
- **单条病害结构化展示**：
  - **ID**: 如 `BRG-202506-001`
  - **类型**: 裂缝 / 剥落 / 泛碱
  - **严重度**: 轻 / 中 / 重 (用绿/黄/红徽章表示)
  - **量化数据（极为关键）**: 长度 1.2m, 宽度 3mm / 面积占比 15%
  - **操作**: `定位` (点击将主视图中心切往该处并高亮 Mask)、`生成维修工单`

### 3. 次核心演示页：病害台账与地图闭环
- **GIS 地图集成**：用深色底图展示桥梁位置（例如杭州复兴大桥、钱塘江大桥），桥梁上标注病害热力图或散点图。
- **工单闭环**：在台账中选择一条病害，点击“生成报告/派单”，模拟输出一份包含无人机原图、AI 遮罩图、量化长宽数据的 PDF 报告格式预览。

## 四、 模拟数据接入标准 (Mock Data JSON)

为了让前端有真实感，负责 UI 开发的 AI 需要预先编写一份结构化的 Mock 数据。
与之前系统不同的是，**位置不仅要给出坐标，还要给出多边形顶点，并加入量化字段。**

示例数据结构参考：

  "bridgeName": "杭州复兴大桥",
  "timestamp": "2026-03-08 14:00:00",
  "frames": [
    {
      "time": 2.5,  // 若为视频则为秒，若为图片组则为序号
      "imageUrl": "/mock-bridges/image_01.jpg",

        {
          "id": "D-001",
          "type": "混凝土裂缝",

          "confidence": 0.98,
          "mask_polygon": [[0.45, 0.50], [0.46, 0.55], [0.48, 0.60], [0.47, 0.50]], // 归一化多边形坐标

## 五、 开发实施步骤与验收清单 (给执行 AI 的指令)

1. **清理旧有代码逻辑**：将项目中原“路况巡检车”、“井盖”、“拥包”等路面概念替换为“无人机桥梁”、“裂缝”、“钢结构锈蚀”等。
2. **全局替换文案**：系统名称、Logo、菜单项全部改为桥梁检测风格。
3. **实现核心检测工作台**：
   - 编写上述 Mock 数据。
   - 使用 React + Canvas/SVG 在占位图片（模拟无人机视角）上渲染出多边形掩膜 (Polygon Masks) 和病害信息标签。
   - 实现**AI 开关对比 (ON/OFF)** 的核心交互逻辑。
   - 实现点击右侧病害列表项，左侧高亮对应掩膜的联动响应。
4. **演示闭环跑通**：能够从“发现病害 (AI Overlay) -> 查看量化数据 -> 转化为维修工单”全流程点击演示。

> **注**：底层真实的深度学习模型训练（YOLO, Mask R-CNN 等）、推流以及真实算力要求，本次前端改造不予关注。只要求**前端展示层看起来已经完美实现了这些功能。**
